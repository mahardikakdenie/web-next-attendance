# Backend Integration & API Requirements

This document outlines the current state of API integration and identified missing endpoints or requirements for the Backend team.

## 🚀 Scope of Issue

The frontend application is currently using a proxy-based security layer (Next.js API Routes) that forwards requests to the backend (`BASE_URL/api/v1/...`). Several features are still using mock data or calling endpoints that return 404 because they haven't been implemented in the backend.

### 🔴 Identified 404 / Missing Endpoints

Based on current service implementation, the following endpoints are expected but may not be fully functional or return 404:

1.  **Attendance & History**
    *   `GET /v1/attendance/history?limit={limit}`
        *   **Description**: Retrieves a list of recent attendance logs for all employees (Admin view) or the current user (User view).
        *   **Query Params**:
            *   `limit` (integer, optional): Number of records to return. Default: 10.
            *   `page` (integer, optional): For pagination.
            *   `status` (string, optional): Filter by "On Time", "Late", "Absent", "On Leave".
        *   **Expected Response Structure**:
            ```json
            {
              "success": true,
              "data": [
                {
                  "id": "att-123",
                  "employee": {
                    "id": "emp-001",
                    "name": "Bagus Fikri",
                    "avatar": "https://example.com/path/to/avatar.jpg"
                  },
                  "date": "2026-04-10",
                  "clock_in": "08:52 AM",
                  "clock_out": "05:30 PM",
                  "location": "Main Office",
                  "status": "On Time",
                  "overtime": "0h 30m"
                }
              ],
              "meta": {
                "total": 124,
                "current_page": 1,
                "last_page": 13
              }
            }
            ```
    *   `GET /v1/attendance/today` (Used in `TodayStatusCard`)
        *   Expected Fields: `clock_in_time`, `clock_out_time`, `status`, `duration`, `date`.
    *   `POST /v1/attendance` (Clock in/out action)
        *   Payload: `{ action: "clock_in" | "clock_out", latitude, longitude, media_url }`.
2.  **Dashboard & Activities**
    *   `GET /v1/activities/recent` (Used in `RecentlyActivity`)
    *   `GET /v1/activities/quick-info` (Used in `QuickInfoCard`)
3.  **Leave & Overtime**
    *   `GET /v1/leaves/balances` (Used in `LeaveBalanceCard`)
    *   `POST /v1/leaves/request` (Form submission)
    *   `POST /v1/overtimes/request` (Form submission)
4.  **Employee Directory**
    *   `GET /v1/employees` (Currently using MOCK_EMPLOYEES)
5.  **Payroll & Calculator**
    *   `GET /v1/payroll/history`
    *   `POST /v1/payroll/calculate`

### 🛠️ Backend Requirements & Specifications

#### 1. Security Headers & Signature (CRITICAL)
The frontend proxy forwards security headers. The backend **MUST** validate these for every request:
*   `X-Timestamp`: Unix timestamp (ms).
*   `X-Request-ID`: UUID generated by the client.
*   `X-Signature`: HMAC-SHA256 hash of `body + timestamp + requestId` using a shared `SIGN_SECRET`.
*   `X-Internal-Secret`: A secret key shared between Next.js and Go/Backend.

#### 2. Authentication & Session
*   **Login**: `POST /v1/auth/login` (Should return `Set-Cookie` for secure session).
*   **Logout**: `POST /v1/auth/logout`.
*   **Me**: `GET /v1/users/me` (Should support `includes` query param for tenant and role data).

#### 3. Media Upload
*   **Endpoint**: `POST /v1/media/upload`.
*   **Content-Type**: `multipart/form-data`.
*   **Requirement**: Return a URL string or an object containing `media_url`.
